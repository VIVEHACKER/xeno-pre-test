# 합격수기 크롤링 및 인터뷰 시스템

## 1. 목적

합격수기를 많이 모으는 것이 목적이 아니다. 합격자가 실제로 어떤 의사결정을 했는지 뽑아내고, 그 규칙을 현재 수험생의 상태에 맞게 재사용하는 것이 목적이다.

## 2. 공개 합격수기 수집 전략

### 2.1 수집 대상

- 고시/자격시험 전문 언론의 합격수기
- 회계사 학원 합격수기 게시판
- 대학 고시반 자료
- 개인 블로그
- 커뮤니티 후기
- 유튜브 자막 또는 요약 가능한 인터뷰

### 2.2 법적/운영 원칙

- 사이트 약관과 robots.txt를 확인한다.
- 원문 전체를 서비스 화면에 재배포하지 않는다.
- 원문은 출처 URL, 제목, 작성자 표시명, 게시일, 추출 신호 중심으로 보관한다.
- 공개 페이지라도 유료 강의 자료, 교재 일부, 문제 원문은 별도 권리 확인 전까지 제외한다.
- 사용자가 보는 화면에는 "구조화된 전략 인사이트"를 제공하고, 필요 시 출처 링크를 제공한다.

## 3. 분석 파이프라인

```text
소스 발견
→ URL 등록
→ 본문 추출
→ 개인정보/광고 제거
→ 시험/연도/과목 분류
→ 학습 이벤트 추출
→ 의사결정 규칙 추출
→ 전략 태그 부여
→ 사람 검수
→ 전략 엔진에 반영
```

## 4. 추출할 필드

### 4.1 프로필

- 전공 여부
- 수험 기간
- 휴학/전업/직장병행
- 초시/재시/다시
- 시작 시점의 베이스

### 4.2 타임라인

- 기본강의 시작과 종료
- 객관식 진입 시점
- 기출 반복 시점
- 모의고사 시작 시점
- 시험 직전 압축 시점

### 4.3 과목별 전략

- 회계학 공부 순서
- 원가관리 포함 여부와 시점
- 세법 말문제/계산문제 분리 방식
- 약점 과목 복구 방식
- 회독 방식

### 4.4 의사결정 규칙

가장 중요한 데이터다.

예:

```text
기본강의 완강 후에도 객관식 정답률이 40% 미만이면 문제 양을 늘리기보다 핵심 개념을 재정렬했다.
```

이 문장은 다음 규칙으로 바뀐다.

```text
IF stage = "post_lecture"
AND subject = "accounting"
AND objective_accuracy < 0.40
THEN assign "core concept rebuild" before "mass problem solving"
```

## 5. 합격자 인터뷰 설계

공개 합격수기는 넓고 얕다. 인터뷰는 좁고 깊어야 한다.

### 5.1 인터뷰 대상

- 최근 3년 내 CPA 합격자
- 회계학 고득점자
- 세법 고득점자
- 비전공 초시 합격자
- 재시 이후 합격자
- 단기간 점수 상승 경험자

### 5.2 인터뷰 방식

- 60분 반구조화 인터뷰
- 실제 기출 2문제를 같이 보며 사고 과정을 말하게 한다.
- 합격수기에는 쓰지 않은 판단 기준을 캐낸다.
- "무엇을 했는가"보다 "왜 그 선택을 했는가"를 묻는다.

### 5.3 핵심 질문

1. 회계학에서 점수가 오르기 시작한 시점은 언제였나?
2. 그 전과 후의 문제풀이 방식은 무엇이 달랐나?
3. 시험장에서 회계학 50문항을 어떤 순서로 풀었나?
4. 어떤 문제는 보자마자 넘겼나?
5. 세법 말문제는 어떻게 암기했고, 계산문제와 어떻게 분리했나?
6. 객관식 문제집을 언제부터 본격적으로 풀었나?
7. 기출을 몇 회독 했고, 회독마다 목적은 무엇이었나?
8. 합격수험생들이 많이 하지만 효과가 낮았던 공부는 무엇인가?
9. 불합격 당시 가장 큰 착각은 무엇이었나?
10. 지금 다시 준비한다면 무엇을 가장 먼저 바꾸겠나?

## 6. 인터뷰 결과의 제품화

인터뷰 답변은 다음으로 변환한다.

- 사용자 유형
- 적용 조건
- 행동 규칙
- 예외 조건
- 추천 강도
- 근거 사례 수

예:

```text
전략명: 회계학 2회전 풀이
적용 조건: 회계학 정답률 55% 이상, 시간 초과율 25% 이상
행동: 1회전에서 계산량 많은 문제를 표시 후 통과, 2회전에서 재진입
예외: 기초 개념 정답률 45% 미만이면 적용 금지
근거: 인터뷰 4명, 공개 합격수기 11건
```
