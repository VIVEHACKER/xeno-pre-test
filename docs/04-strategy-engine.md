# 합격 전략 엔진 설계

## 1. 역할

전략 엔진은 공부 콘텐츠를 추천하는 기능이 아니다. 사용자의 현재 상태에서 합격 가능성을 가장 크게 끌어올리는 다음 행동을 결정하는 시스템이다.

입력:

- 사용자 목표 시험일
- 하루 공부 가능 시간
- 회계학/세법개론 정답률
- 시간 초과율
- 오답 원인
- 개념 숙련도
- 최근 수행률
- 합격수기/인터뷰 기반 전략 규칙

출력:

- 오늘의 공부 처방
- 이번 주 목표
- 우선순위 개념
- 풀 문제와 건너뛸 문제
- 리스크 경고
- 검증 지표

## 2. 사용자 상태 분류

### 2.1 단계

- 입문: 기본 개념 이해 전
- 기본강의 이후: 개념은 들었지만 객관식 전환 전
- 객관식 진입: 문제풀이 시작
- 기출 회독: 반복과 약점 보완
- 모의고사: 시간 운영과 실전 적응
- 파이널: 압축, 암기, 실수 제거

### 2.2 리스크 축

- 개념 부족
- 계산 실수
- 조건 누락
- 시간 초과
- 세법 휘발
- 회독 지연
- 과목 편중
- 계획 수행률 저하

## 3. 중요도 점수

개념 중요도는 단순 출제 빈도가 아니다.

```text
importance_score =
  출제 빈도 * 0.25
+ 최근 출제성 * 0.15
+ 배점 기여도 * 0.15
+ 사용자 약점도 * 0.25
+ 변형 가능성 * 0.10
+ 합격수기/인터뷰 강조도 * 0.10
```

초기에는 규칙 기반으로 시작하고, 사용자 데이터가 쌓이면 가중치를 학습한다.

## 4. 합격 리스크 점수

합격확률이라는 표현은 조심스럽게 쓴다. 초기 제품에서는 "합격 리스크"가 더 정확하다.

```text
risk_score =
  과락 위험
+ 시간 초과 위험
+ 핵심 개념 공백
+ 계획 수행률 저하
+ 시험일까지 남은 기간 부족
+ 최근 2주 점수 정체
```

출력 예:

```text
현재 리스크: 높음
주된 원인: 회계학 시간 초과와 세법 말문제 휘발
이번 주 목표: 회계학 시간 초과율 34%에서 25% 이하로 감소
```

## 5. 처방 생성 로직

### 5.1 예시 규칙

```text
IF subject = "accounting"
AND accuracy >= 0.55
AND time_overrun_rate >= 0.30
THEN assign "two-pass solving drill"
```

```text
IF subject = "tax"
AND verbal_question_accuracy < 0.50
AND calculation_question_accuracy >= 0.60
THEN assign "daily verbal recall block"
```

```text
IF days_until_exam < 60
AND concept_mastery < 0.45
THEN prevent "new lecture expansion"
AND assign "high-yield concept recovery"
```

## 6. 하루 처방 구조

```text
오늘의 목표
1. 핵심 개념 3개 복습
2. 문제 세트 24문항
3. 시간 제한 풀이 1회
4. 오답 원인 분류
5. 내일로 넘길 유형 결정
```

각 처방은 반드시 검증 지표를 가진다.

예:

- 회계학 금융자산 문제 12개 중 8개 이상 정답
- 평균 풀이시간 95초 이하
- 조건 누락 오답 2개 이하
- 세법 말문제 30개 중 재오답 5개 이하

## 7. 전략 엔진 MVP

첫 버전은 아래 로직으로 충분하다.

1. 사용자 입력 기반 진단
2. 문제풀이 로그 기반 약점 계산
3. 개념 중요도 점수 계산
4. 합격수기 규칙 매칭
5. 주간 계획 생성
6. 다음 진단에서 계획 수정

## 8. 금지할 출력

- "무조건 합격"
- "이 문제만 보면 된다"
- "AI 예측 적중"
- 근거 없는 특정 강사/교재 추천
- 사용자 상태와 무관한 일반론

## 9. 좋은 출력

```text
이번 주에는 세법 전체 회독보다 부가가치세 계산 구조와 법인세 익금/손금 판단을 먼저 잡는 편이 좋습니다.
이유: 최근 오답 18개 중 11개가 계산식이 아니라 과세표준 산입 여부 판단에서 발생했고, 시험일까지 84일 남아 있어 지금은 암기량 확대보다 판단 기준 압축이 점수 상승에 더 직접적입니다.
```
